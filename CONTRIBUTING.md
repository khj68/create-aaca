# Contributing to AACA

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes
4. Run validation (`npm run validate`)
5. Commit your changes
6. Push to the branch
7. Open a Pull Request

## Development Setup

```bash
git clone https://github.com/khj68/aaca.git
cd aaca
npm install
```

## Areas of Contribution

- **New language templates** (`templates/{language}/`)
- **Schema improvements** (`schema/`)
- **CLI enhancements** (`cli/`)
- **Documentation** (`docs/`)
- **Example projects** (`examples/`)

## Guidelines

- Every new directory must include a `CONTEXT.md`
- Follow the existing manifest formats
- Add tests for CLI changes
- Update relevant documentation
